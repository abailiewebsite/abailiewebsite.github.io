About Manulife

Manulife, founded 132 years ago in 1887, is a Canadian insurance and finance company that provides services all over the world. The company operates in Canada and Asia, under the name Manulife and under the John Hancock name in the United States. The name "Manulife" is short for The Manufactures Life Insurance Company, and the companies first president was the first prime minister of Canada, John A. Macdonald. Headquatered in Toronto, Manulife is the largest insurance company in Canada and with its 34,000 employees they service over 26 million customers worldwide.


Learning Goals

I wanted to strengthen my knowledge of all things web development. Specifically, becoming proficient at developing with ReactJS, but also doing it correctly along the way. This meant all the React components I created would have to be put through unit testing and pass with 100% code coverage.

React is a powerful tool. It can be used to develop webistes, or both iOS and Android apps without the need to write the same app twice. So I made it a goal of mine to learn.

Action Plan

Study, research, and practice while putting in an honest work ethic.

Job Description

During my time at Manulife, I held the position of Full Stack Software Engineer. This was my first full-stack role, which I was excited for. The team I was placed with at Manulife was called the Capability Squad of the Individual Insurance Engineering Group. This team has been working on a project for the last 2 years to what is projected to be a 4-year project. In short, the project is to take the exisiting website that Manulife's insurance advisors use to sign up clients, called Advisor Portal, and rebuild it with modern technologies while making much-needed improvements along the way. My team decided to use React to build the new website and use NodeJS to develop the micro-services in our backend.

My typical day started with a skype conference call between all members of the Capability Squad at 9:15 AM, where we would talk about what we did the day before, what we had planned for today, and if we had anything blocking us from getting our work done, this was essentially our daily stand-up. Every two weeks we would have a skype conference call between all the teams where our manager would discuss company news, policy updates, and training for the full-time employees. In this call would be teams from Kitchener, Waterloo, and Toronto.

My work at Manulife varied greatly since I was full-stack. I started in the front-end, helping build simple web pages using HTML / CSS and the skills I picked up from my last co-op. I then moved into adding components into our React pages, while also writing unit tests using Jest before a merge request into our working branch could be approved. I then moved into the back-end, where I was tasked with creating Jenkins pipelines. These pipelines would be used to trigger a Sonarqube check of the code automatically whenever new code got pushed. These pipelines helped control the quality of our code. I eventually moved onto helping build our API. Building routes that would grab data from our database package it nicely, and send it to our front-end. This is where I got exposure to Swagger, which I found very nice to use.

There where a lot of learning curves during this co-op. I essentially had to learn everything I was working on as I have not done much of it before. However, I put in the time and work and it has paid off. As I result, I have become a better developer because of it.
